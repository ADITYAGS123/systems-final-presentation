3. Linux Boot: Power-On to Desktop

Basic Flow

Power On
   ↓
BIOS / UEFI
   ↓
POST (Hardware Check)
   ↓
Boot Device Selection
   ↓
Bootloader (GRUB2 / LILO)
   ↓
Kernel
   ↓
systemd (init)
   ↓
Services + Desktop

Step-by-Step Explanation

  1. Power On
    CPU starts execution from a fixed memory location (reset vector).
  2. BIOS / UEFI
    Firmware initializes hardware (CPU, RAM, keyboard, etc.).
    • BIOS: Uses MBR (Master Boot Record), limited to ~2TB disks
    • UEFI: Uses GPT, faster boot, supports Secure Boot
  3. POST (Power-On Self Test)
    Checks if hardware components are working properly.
    If any issue is found → error message is shown.
  4. Boot Device Selection
    Firmware selects boot device based on priority:
    Hard disk → USB → CD/DVD (can be configured in BIOS settings)
  5. Bootloader
    Located differently depending on system:
    • BIOS → stored in MBR
    • UEFI → stored as .efi file in EFI partition

    Functions:

    • Locate OS kernel on disk
    • Load kernel into memory
    • Start kernel execution

    Types:

    • GRUB2: Modern, widely used, supports multiple OS
    • LILO: Older, mostly obsolete
  6. Kernel
    After GRUB loads it:
    • Decompresses itself into memory
    • Initializes hardware
    • Loads device drivers & kernel modules
    • Starts background processes
  7. Init Process (systemd)
    First user-space process (PID 1).
    • Handles system startup
    • Mounts file systems
    • Starts all services
  8. Services & System Initialization
    systemd starts:
    • Networking
    • Sound
    • Power management
    • Other background services
  9. Login & Desktop
    • User login (GUI or terminal)
    • Desktop environment loads (GNOME/KDE etc.)

BIOS vs UEFI (Quick Comparison)

BIOS

  • Older system
  • Uses MBR partition
  • Limited disk size (~2TB)
  • Slower boot
  • Less secure

UEFI

  • Modern replacement
  • Uses GPT partition
  • Supports large disks
  • Faster boot
  • Secure Boot support

Key Insight

Booting is a chain process:
Each stage prepares and hands control to the next stage until the system becomes usable.

Exam Tip

Always write this order:
Power → BIOS/UEFI → POST → Bootloader → Kernel → systemd → Services → Desktop
