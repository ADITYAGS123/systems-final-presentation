1 & 2. Binaries and Libraries — The Building Blocks

A binary is a file of machine instructions the CPU can execute. It is produced by compiling source code and linking object files and libraries into an executable format (e.g., ELF on Linux).

From Source to Binary

Source (.c/.cpp/.rs)
   │  compile
   ▼
Object Files (.o)
   │  link (+ startup + symbols)
   ▼
Executable (ELF/EXE) → entry point → OS loads → CPU runs

What is a Library?

A library is precompiled code that programs reuse. Two main kinds on Linux:

  • Static .a — code is copied into the executable at link time (larger binary, self‑contained).
  • Shared .so — loaded by the dynamic linker at program start (smaller app, shared in RAM, versioned).

How Binaries & Libraries Work Together

Object Files (.o) & Static Libs (.a) --(link)--> Final Binary
Final Binary --(runtime)--> Dynamic Linker loads Shared Libs (.so)

Why it matters: impacts startup time, memory usage, portability, and security updates.

3. Linux Boot: Power-On to Desktop

  1. Power on → CPU reset vector jumps into firmware.
  2. BIOS/UEFI initializes hardware, performs POST, and locates a bootable device.
  3. Bootloader (GRUB) presents a menu, loads the Linux kernel and initramfs, passes boot params.
  4. Kernel decompresses, initializes drivers, mounts temporary root (initramfs), finds real root FS, starts PID 1.
  5. systemd (PID 1) mounts filesystems, starts services (udev, network, logs), and launches a display manager.
  6. Display server & Desktop → Login via GDM/SDDM/LightDM; Wayland/Xorg session starts GNOME/KDE/XFCE.
Power → Firmware → Bootloader → Kernel+initramfs → systemd → Display Manager → Desktop

4. Embedded Systems — Purpose‑Built Computing

An embedded system combines hardware + firmware (and often an RTOS or embedded Linux) to perform a dedicated function under power, cost, and timing constraints.

Typical Stack

Application / Control Logic
RTOS or Embedded Linux
Drivers (GPIO, SPI, I2C, UART)
MCU/MPU + RAM/Flash + Peripherals

Common Types

  • Standalone controllers (appliances)
  • Real‑time systems (ABS, pacemakers)
  • Networked/IoT devices (routers, wearables)
  • Embedded Linux devices (TVs, gateways)

Key Traits

  • Deterministic timing (often)
  • Small memory & storage footprint
  • Optimized boot & power usage
  • High reliability, long life cycles
Contrast with PCs: general‑purpose, larger RAM/CPU, rich GUIs, frequent updates.

Appendix: Quick Reference & Speaker Notes

Key Terms

  • ELF: Executable and Linkable Format (Linux binaries)
  • initramfs: temporary root FS used during early boot
  • PID 1: first user‑space process (systemd)
  • MCC: Merchant Category Code (not relevant here but often asked in payments)

Demo Ideas

  • Run ldd /bin/ls to show shared libs loaded by a binary
  • Show GRUB menu & kernel parameters
  • Use dmesg to skim kernel init logs
  • On a microcontroller, blink LED (bare‑metal vs RTOS)
